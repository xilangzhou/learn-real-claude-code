# s06: Context Management Stack

## 为什么需要这一章

长时间运行的 agent 面对的不是一个上下文问题，而是很多问题：超大工具输出、过期历史、以及模型自己意识到需要重置的时刻。

## 核心机制

- 在大结果压垮历史之前先裁剪
- 用滚动摘要维持连续性
- 允许模型显式请求 manual compact
- 把这些看成不同干预，而不是一个按钮

## 这一章如何映射到 Python 代码

`agents/s06_context_compact.py` 用一个 context manager 对象承载策略，而不是把单次压缩硬编码进循环。

## 这里刻意简化了什么

真实产品可能还有更多 collapse 路径和 budget 启发式。教学版已经是策略栈，但仍然是简化后的策略栈。

## 建议你自己试一试

- 先打开 `agents/` 里本章对应的 Python 文件，观察新增类、工具或运行时状态。
- 给模型一个必须用到本章机制的任务，而不是只问概念定义。
- 关注“状态是存在哪儿、谁负责更新、父子/前后台/多 worker 如何看到它”。

## 它如何连接到下一章

当历史管理能力建立后，运行时就可以进一步把持久工作状态彻底移出 transcript。
