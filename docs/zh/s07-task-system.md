# s07: Persistent Task Runtime

## 为什么需要这一章

聊天中的 checklist 很有用，但会随会话一起消散。严肃工作跟踪必须拥有自己的存储与生命周期。

## 核心机制

- 把任务持久化到消息历史之外
- 支持 task create/get/list/update
- 把任务视为 durable runtime state
- 在概念上与 session todo 明确分离

## 这一章如何映射到 Python 代码

`agents/s07_task_system.py` 把持久状态写入专门运行时目录，让 task plane 不再依赖聊天窗口。

## 这里刻意简化了什么

教学版采用紧凑的文件存储。重点在于“持久化”和“关注点分离”，而不是产品级任务编排。

## 建议你自己试一试

- 先打开 `agents/` 里本章对应的 Python 文件，观察新增类、工具或运行时状态。
- 给模型一个必须用到本章机制的任务，而不是只问概念定义。
- 关注“状态是存在哪儿、谁负责更新、父子/前后台/多 worker 如何看到它”。

## 它如何连接到下一章

有了持久状态之后，运行时就可以开始拥有那些前台 loop 继续往前走时仍会持续的工作。
