# s01: The Agent Loop

## 为什么需要这一章

一个 coding agent 的起点不是规划器、不是多 agent 团队，也不是工作流图，而是一个能观察和行动的循环。

## 核心机制

- 把用户消息追加到历史
- 携带工具定义发起模型调用
- 如果模型请求工具，就执行并回写结果
- 如果没有工具调用，就返回最终回答

## 这一章如何映射到 Python 代码

`agents/s01_agent_loop.py` 只保留可见教学外壳，把可复用循环辅助逻辑收进 `agents/_shared.py`。

## 这里刻意简化了什么

真实系统还会处理 streaming、重试、权限、token 预算和中断恢复。这一章有意忽略这些复杂度，只保留循环不变量。

## 建议你自己试一试

- 先打开 `agents/` 里本章对应的 Python 文件，观察新增类、工具或运行时状态。
- 给模型一个必须用到本章机制的任务，而不是只问概念定义。
- 关注“状态是存在哪儿、谁负责更新、父子/前后台/多 worker 如何看到它”。

## 它如何连接到下一章

当循环成立之后，下一步真正的问题不是如何重写循环，而是如何在不碰循环的前提下增加能力。
