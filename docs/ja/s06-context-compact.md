# s06: Context Management Stack

## なぜこの章が必要か

長時間走るエージェントが抱える文脈問題は一つではありません。巨大なツール出力、古い履歴、モデル自身がリセットを必要とする瞬間など複数あります。

## コアメカニズム

- 巨大な結果が履歴を支配する前に刈り込む
- continuity のために rolling summary を維持する
- manual compact を明示的に許す
- これらを一つのボタンではなく別個の介入として扱う

## Python コードへの対応

`agents/s06_context_compact.py` は一回限りの圧縮をループに埋め込まず、context manager が方針を持つ形にします。

## 意図的に単純化している点

実製品にはさらに多くの collapse path や budget heuristic があり得ます。教材版はスタックですが、なお簡略化されたスタックです。

## 試してみること

- `agents/` の対応する Python ファイルを開き、追加された class、tool、runtime state を確認する。
- この章の新機構を実際に使わないと進まないタスクを与える。
- 状態がどこに保存され、誰が更新し、誰から見えるかを追う。

## 次章へのつながり

履歴管理ができるようになると、永続作業状態を transcript の外へ完全に出す準備が整います。
