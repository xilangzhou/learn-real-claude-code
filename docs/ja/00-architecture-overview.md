# アーキテクチャ概要

本コースは `claude-code` 系の実運用から重要な runtime の考え方を抜き出し、`agents/` の小さな Python ハーネスに落としています。章立ては `learn-claude-code` に寄せてありますが（一章に主メカニズム一つ）、パスやディスク上の约定はこのリポジトリ固有です（特に `.lrcc/` 以下の状態）。

## 何を学ぶか

モデルは推論できますが、自動ではディスクを読んだりコマンドを実行したりはしません。エージェントの出発点は **ループ** である：リクエスト → モデルがツールを呼ぶかも → 実行 → 結果を足す → モデルがツールを求めなくなるまで繰り返す。

その骨格の上に、各章が足していくのは：構造化ツール、セッション todo、委任モード、オンデマンド skill、コンテキスト整理、永続タスク、バックグラウンド通知、チーム inbox、プロトコル状態、自律ポーリング、タスクと worktree の紐づけである。「動く」から「協調と隔離まで」への一続きの道であり、機能の寄せ集めではない。

## リポジトリの読み方

| パス | 役割 |
|------|------|
| `agents/s*.py` | 章ごとの最小実行ハーネス |
| `agents/_shared.py` | 共有の `run_loop`、`ToolRegistry`、基本ツール |
| `docs/{en,zh,ja}/` | 各言語の章本文（Web チュートリアルと同源） |
| `web/` | コース UI と可視化 |

章を動かすとき：リポジトリルートで `python agents/sXX_....py`。環境変数に `MODEL_ID`（任意で `ANTHROPIC_BASE_URL`）を設定する。
